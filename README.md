@@@Component란?
=>컴포넌트란? 프로그래밍에 있어 재사용이 가능한 각각의 독립된 모듈을 뜻한다
=>하드웨어처럼 독립적인 기능을 수행하고 추후에 교환될 수 있도록 해야한다! 또한
컴포넌트의 세부사항은 겉으로 드러나면 안되는 것이며, 일부러 끄집어 내려는 시도조차 하지말아야 된다. 필요한 것은 단지 해당 컴포넌트를 쓰기 위해 제공해주는 일종의'장치'가 필요하다. 예시로 핸드폰의 베터리 수명이 다 했을 경우
베터리를 새로 구입하여 베터리만 교체했을 것이다! 이때 핸드폰 단자와 베터리 단자가 맞물려야 제 기능을 발휘하는 데 이때 베터리 단자가 앞서 말했던 일종의 '장치'가된다!
=>시스템을 유지보수하는데 있어 교체가능한 부품이다

React 컴포넌트는 선언하는 방식에따라
클래스 기반의 컴포넌트 와
함수 기반의 컴포넌트로 구분 할 수 있다!

@@@import와 export란??
import문은 다른 곳에서 정의된 코드를 App.js에서 사용할 수 있도록 불러오는 구문!
export문은 선언한 App컴포넌트를 다른 모듈에서불러와 사용할 수 있도록 내보내는 구문!

@@@중요?!)
React에서 모듈은 크게 두 가지 유형으로 구분 할 수 있다
1.하나의 컴포넌트만 선언 되어 있는 모듈
2.복수의 컴포넌트가 선언되어 있는 라이브러리 형태의 모듈
=> 만약 2번처럼 복수의 컴포넌트를 사용할 경우에는? 선언부에 export를 붙여준다!

@@@props(properties)란?
=>컴포넌트에 데이터를 전달하고자 할 때 사용하며, 이렇게 전달된 값은 변수를 통해 참조할 수 있다
=>HTML과 같이 위에서 부터 아래로 읽힌다! props값은 해당 컴포넌트를불러와 사용하는 부모 컴포넌트에서 설정할 수 있다!
******예시*******
const Laptop = (props) => {
  return <h1>내 노트북은 {props.brand} 노트북입니다.</h1>;
};

const App = () => {
  return <Laptop brand="Samsung" />;
};

export default App;
